# REVIEW - Open Model Financial Services

## Project Decisions

### Charting Library Selection

**Decision**: Use Plotly.js (not ECharts)

**Rationale**:

- Plotly has better Python integration for data scientists
- Easier to generate from Python scripts
- Wide adoption in financial analysis
- Mature ecosystem with good documentation

**Trade-offs**:

- ECharts offers more animation options
- Plotly bundle size is larger
- Some customizations require more code

## Technical Architecture

### File Structure

```
open_model_financial_services/
├── _agents/              # AI agent configurations
│   └── skills/          # Specialized skills
├── guides/              # Documentation
├── notebooks/           # Jupyter notebooks for analysis
├── plans/               # Project planning
├── scripts/             # Data processing and report generation
│   ├── generate_html_report.py      # Main dashboard generator
│   ├── generate_html_report_lib.py   # Shared library functions
│   └── *.py            # Data cleaning scripts
├── .gitignore          # Excludes data/ and output/
└── README.md           # Project documentation
```

### Data Flow

1. Raw data pulled from external sources (ONS API, etc.)
2. Data cleaned and processed via scripts in `scripts/`
3. HTML dashboard generated by `generate_html_report.py`
4. Output deployed to GitHub Pages

## Implementation Notes

### generate_html_report_lib.py

- Data loading functions

Centralized color palette for consistent visualization:

- Primary Red: `#ef4444`
- Primary Blue: `#3b82f6`
- Success Green: `#22c55e`
- Warning Amber: `#f59e0b`
- Neutral Gray: `#6b7280`

### GitHub Pages Deployment

- Automatic deployment via GitHub Actions
- Service worker for offline capability
- Custom 404 page for better UX

## Lessons Learned

### What Worked Well

1. **Separate data and output from git** - Prevents large file issues
2. **Modular script structure** - generate_html_report_lib.py enables code reuse
3. **Service worker** - Provides offline viewing capability

### Areas for Improvement

1. Add unit tests before implementing features
2. More comprehensive error handling
3. Data refresh automation
